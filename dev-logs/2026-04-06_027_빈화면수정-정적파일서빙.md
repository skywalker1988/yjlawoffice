# 빈 화면 수정 — 프로덕션 정적 파일 서빙 추가

**일시**: 2026-04-06  
**커밋**: `0be6bf6`

## 문제
- 홈페이지가 아예 빈 화면으로 표시됨
- 원인: 백엔드(Express 5)에서 프론트엔드 빌드 파일을 서빙하지 않음
- Vite dev 서버(5173)가 아닌 백엔드(5000)에 직접 접속 시 빈 화면

## 수정 내용
- `backend/index.js`에 프로덕션 정적 파일 서빙 추가
  - `express.static(frontendDist)` — 빌드된 프론트엔드 파일 서빙
  - SPA 폴백 미들웨어 — API 외 모든 GET 요청에 index.html 반환
  - Express 5 호환 — `app.get("*")` 대신 `app.use()` 미들웨어 사용
  - Windows 경로 호환 — `path.resolve().replace(/\\/g, "/")` 적용

## 트러블슈팅 과정
1. 프론트엔드 코드 리뷰 → 모든 페이지에 기본값(defaults) 있어 데이터 문제 아님
2. Vite build 성공 (에러 없음) → 빌드 문제 아님
3. Express 5의 `app.get("*")` PathError 발생 → 미들웨어 방식으로 변경
4. `path.resolve()`의 백슬래시 경로 → `express.static` 작동 안 함 → 포워드 슬래시 변환
5. 노드 프로세스 충돌 → 전체 프로세스 종료 후 재시작으로 해결

## 현재 상태
- 백엔드 `http://localhost:5000` — 프론트엔드 + API 모두 정상 서빙
- 프론트엔드 dev `http://localhost:5173` — 개발 모드 정상 동작
