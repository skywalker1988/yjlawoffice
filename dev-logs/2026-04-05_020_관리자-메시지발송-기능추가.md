# 2026-04-05 | 관리자 메시지 발송 기능 (SMS + 이메일)

## 프롬프트
> 관리자페이지에서 문자/이메일 보내는 기능을 추가하고 싶어. 사무실 전화번호를 발신번호로 해서 문자를 보내고 싶고, 문자 보내는 시기, 문자 내용 등을 관리자 페이지에서 관리하고 싶어.

## 추가 요청
- 고객한테 문자/이메일이 언제 발송되었는지 기록 남기기
- 텍스트 깨지지 않도록 UTF-8 인코딩 처리
- 샘플 문자 템플릿 미리 만들기
- SMS + 이메일 동시 발송 기능

## 구현 내용

### DB 스키마 (2개 테이블 추가)
- **message_templates**: 재사용 가능한 메시지 템플릿 (SMS/이메일 채널별)
- **message_logs**: 발송 이력 (발송 시각, 성공/실패 상태, 에러 메시지 포함)

### 백엔드 서비스 모듈
- `backend/lib/sms-service.js` — CoolSMS SDK 래퍼 (SMS/LMS 자동 전환)
- `backend/lib/email-service.js` — Nodemailer Gmail SMTP (UTF-8 인코딩 명시)

### 백엔드 API 라우트 (`/api/sb/messages`)
- 템플릿 CRUD: GET/POST/PATCH/DELETE /templates
- 메시지 발송: POST /send (SMS/이메일/동시 발송)
- 발송 이력: GET /logs, DELETE /logs/:id
- 시간당 50건 발송 제한 (rate limiter)

### 프론트엔드 (`AdminMessages.jsx`)
3탭 구성:
1. **템플릿 관리** — 템플릿 CRUD, 채널별 분류, 플레이스홀더 안내
2. **메시지 발송** — 상담 고객 선택 → 템플릿 또는 직접 입력 → SMS/이메일/동시 발송
3. **발송 이력** — 날짜/채널/상태 필터, 페이지네이션

### 샘플 템플릿 (9개)
- SMS 6개: 접수확인, 일정확정, 일정변경, 리마인더, 완료감사, 서류요청
- 이메일 3개: 접수확인, 일정확정, 완료감사 (HTML 서식 포함)

### 플레이스홀더
- `{name}` — 수신자 이름
- `{date}` — 오늘 날짜 (한국어 형식)
- `{category}` — 상담 분야 (민사, 형사 등)
